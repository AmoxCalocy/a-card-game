# 一人旅途 · 基础版实施计划（给 AI 开发者的分步指令）
> 目标：先完成单机基础玩法（旅途探索 + 卡牌战斗 + 伙伴与资源 + 基础事件），再迭代到帝国阶段。

## 阶段 0：项目基线
1. 初始化 Unity 6 LTS URP 2D 项目，启用 Input System、TextMeshPro、Addressables。  
   - **测试**：启动空场景无报错；Addressables Analyze 通过。
2. 建立 Git + Git LFS；创建分支策略（main/dev/feature）。  
   - **测试**：首次 commit 成功；大文件（>50MB）自动走 LFS。
3. 配置 CI（GitHub Actions + unity-builder）打包 Windows 开发版。  
   - **测试**：PR 触发自动构建成功并产出可下载工件。

## 阶段 1：数据与架构骨架
4. 定义 ScriptableObject 模板：卡牌、敌人、事件、伙伴、遗物、资源表。  
   - **测试**：在 Inspector 可创建各类型资产，无序列化警告。
5. 搭建全局 GameContext（单例/服务定位）管理资源、卡池、事件池、当前旅途状态。  
   - **测试**：进场景后能读取初始卡池与资源数值且在 UI 调试面板显示。
6. 建立事件总线或 UniRx 事件流，约定消息类型（卡牌抽取、资源变更、节点选择等）。  
   - **测试**：触发资源变更时 UI 调试面板实时更新，无空引用。

## 阶段 2：世界与节点流程
7. 实现节点式大地图生成（若干分支节点：战斗/事件/补给/首领）。  
   - **测试**：生成 10+ 节点且至少 2 条分支路径，节点类型分布符合权重。
8. 旅途推进：点击节点进入对应场景，完成后移动到下一个节点，消耗粮食。  
   - **测试**：粮食归零后禁止继续前进并显示提示。
9. 危机值与随机事件挂钩：每前进一节点增加危机值，超过阈值强制触发灾害事件。  
   - **测试**：手动将危机值拉高触发灾害，事件类型正确出现。

## 阶段 3：战斗核心循环
10. 战斗入口：从战斗节点生成敌方队列并载入战斗场景。  
    - **测试**：进入战斗时敌人列表与节点配置一致。
11. 回合流程：玩家先手，伙伴共享能量，出牌→弃牌→敌方行动→抽牌补满。  
    - **测试**：每回合能量重置；弃牌与抽牌计数正确，卡堆不会越界。
12. 牌类型与效果框架：攻击、防御、策略、后勤、战术；可挂状态与数值。  
    - **测试**：创建 5 张示例牌，按描述生效且可叠状态（如流血）。
13. 敌人 AI 意图：下回合意图显示（攻击/防御/掠夺）。  
    - **测试**：意图图标与实际行动一致；变更时 UI 及时更新。
14. 战斗结算：胜利掉落卡牌/资源，失败扣资源并可能伙伴受伤。  
    - **测试**：手动胜/负后资源与伙伴状态按规则变化。

## 阶段 4：伙伴与队伍
15. 伙伴招募事件：事件结果可将伙伴加入队伍（最多 4 上阵）。  
    - **测试**：招募后卡组自动加入该伙伴专属基础卡。
16. 伙伴特质与忠诚度：特质影响技能检定；忠诚度低时有离队提示。  
    - **测试**：模拟忠诚度降低，触发离队警告弹窗。
17. 队伍编排 UI：拖拽调整上阵顺序，影响战斗站位/技能。  
    - **测试**：更换顺序后战斗中顺序生效。

## 阶段 5：资源与经济
18. 资源系统：粮食、财富、声望、医疗物资；统一接口增减与日志。  
    - **测试**：调用接口时日志记录变化，数值不会出现负数（除危机值）。
19. 交易节点：简单买卖，价格受声望影响。  
    - **测试**：声望提高后同物品价格下降；买卖后库存与资源正确更新。
20. 补给与治疗：在补给节点可用财富换粮，医疗物资治愈伙伴伤病/疾病牌。  
    - **测试**：治疗后负面状态消失且资源扣减正确。

## 阶段 6：事件系统
21. 事件模板：支持多选项（战斗、检定、支付、牺牲卡牌）。  
    - **测试**：事件配置三种不同解决方式，均能落地到结果（奖励/惩罚）。
22. 技能检定：基于伙伴特质与声望；成功/失败有不同分支。  
    - **测试**：修改声望后检定成功率变化，结果分支正确。
23. 灾害事件：瘟疫/劫匪/自然灾害；可污染卡组（诅咒/疾病牌）。  
    - **测试**：触发灾害后卡组新增对应负面牌，显示来源标记。

## 阶段 7：UI/UX 与可用性
24. 主 HUD：展示资源、危机值、当前节点信息、抽弃牌堆计数。  
    - **测试**：切换场景资源数值保持一致，UI 无跳变。
25. 卡牌提示：鼠标悬停/长按显示详细说明与关键词解释。  
    - **测试**：所有示例牌的提示文本与实际效果一致。
26. 快速教程：首场旅途的分步引导（点击高亮、简短文案）。  
    - **测试**：新存档自动进入教程；完成后可在设置关闭。

## 阶段 8：存档与设置
27. 存档系统：自动/手动槽各 1，保存旅途状态、牌堆、伙伴、节点位置。  
    - **测试**：存档后重启游戏能恢复到相同节点与资源。
28. 设置菜单：音量、文本速度、语言（占位）、键位/手柄映射。  
    - **测试**：调整设置后立即生效且写入本地配置。

## 阶段 9：平衡与内容种子
29. 初版内容包：40 张基础牌、10 敌人、20 事件、8 伙伴、8 遗物。  
    - **测试**：完成一条 10 节点路线，平均通关时长 30-40 分钟。
30. 简易数值校验：敌人血量=队伍平均输出*1.2；掉落随章节递减 10%。  
    - **测试**：跑 5 次自动战斗模拟，胜率落在 40%-60% 区间。

## 阶段 10：稳定性与打包
31. 性能与内存：Profiler 监测战斗场景帧率>60（1080p PC），内存峰值 <1.5GB。  
    - **测试**：在高频出牌与多状态叠加下仍满足指标。
32. 异常处理：缺资源、卡组为空、Addressables 失败等，提供友好提示。  
    - **测试**：拔网模拟热更失败，弹出重试/离线继续选项。
33. 打包与分发：CI 生成内测包，自动附带版本号与变更日志。  
    - **测试**：下载内测包能直接运行并显示正确版本标识。

后续扩展（帝国阶段、多周目、法令卡、联机等）待基础版完成后再立新增里程碑。
